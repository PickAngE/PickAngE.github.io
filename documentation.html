Command Documentation

Welcome to the complete Kernel Bot documentation. Here you will find detailed information about all available commands, organized by category for easy navigation.

Quick Start

All commands use Discord's slash command system. Simply type / followed by the command name to get started.

Support Commands

Ticket System

Creates a comprehensive support ticket system that allows users to get private help from staff members. This system includes automatic archiving and role management.

/ticket setup #category @support_role #archive_channel @mention_support
Parameters:
• category: Channel category for tickets
• support_role: Role to ping for new tickets
• archive_channel: Where closed tickets go
• mention_support: Auto-mention support staff

Logs Commands

setlog

Configure the primary logging channel where the bot will send all server activity logs and important notifications.

/setlog #channel
Note: Requires admini permissions

Showlog Config

Display the current logging configuration for your server, including which events are being logged and where.

/showlogconfig
Info: Shows all active log settings

Logconfig

Interactive configuration menu to easily set up and customize all logging options for your server.

/logconfig
Features: Interactive setup wizard

Utility Commands

User Info

Get comprehensive information about any user in the server, including join date, roles, and activity status.

/info @user
Shows: Avatar, roles, join date, account age

Reaction Role

Set up reaction-based role assignment on any message. Users can react to get or remove roles automatically.

/reactionrole message_id emoji @role
Tip: Right-click message → Copy ID to get message_id

Mention User

Configure automatic user mentions when new channels are created, useful for notification systems.

/mentionuser adduser @user
Usage: Automatically mentions users in new channels

Voice Chat Commands

Move

Move a specific user from their current voice channel to another voice or stage channel of your choice.

/move @user #destination_channel
Requirement: User must be in a voice channel

Moveall

Transfer all users from one voice or stage channel to another channel in bulk. Perfect for event management.

/moveall #source_channel #destination_channel
Effect: Moves all users from source to destination

Moveall2

Move all currently connected voice and stage channel members to a single specified channel.

/moveall2 #destination_channel
Scope: Affects all voice-connected users

Help Commands

Help

Display a comprehensive list of all available commands that regular users can access, with brief descriptions.

/help
Access: Available to all server members

Adminhelp

Special help command that shows administrative commands and advanced features reserved for server admin.

/adminhelp
Restricted: Admin permissions required

Usage Tips & Best Practices

Permissions

Ensure Kernel Bot has the necessary permissions for each command. Administrative commands require proper role hierarchy.

Performance

Commands execute instantly. For bulk operations like moveall, there might be a slight delay to prevent rate limiting.

Customization

Many commands can be customized for your server's specific needs. Contact support for advanced configurations.
